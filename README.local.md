# prakse_Sergejs_Fjodorovs

Локальный backend-проект на Node.js и Express для регистрации и получения пользователей с хранением данных в MySQL.

## Что делает программа

Приложение поднимает HTTP API и предоставляет:

- Проверку работоспособности сервера через endpoint health — GET /health возвращает JSON с статусом 'ok' и временной меткой, позволяя убедиться, что сервер доступен и работает.
- Регистрацию пользователя через POST users — принимает email и пароль, валидирует их, хеширует пароль с bcrypt и сохраняет пользователя в БД, возвращая ID нового пользователя.
- Валидацию входных данных пользователя (email и пароль) — проверяет формат email по regex и минимальную длину пароля (8 символов), возвращая ошибки валидации если данные некорректны.
- Хеширование пароля перед сохранением в базу данных — использует bcrypt с 10 salt rounds для безопасного хранения паролей, гарантируя, что реальные пароли не хранятся в открытом виде.
- Получение списка пользователей через GET users с пагинацией и фильтром по email — возвращает пользователей постранично (параметры page, limit) и может отфильтровать по email через поиск по части адреса.
- Централизованную обработку ошибок (включая конфликт по дублирующемуся email) — единый error middleware перехватывает все ошибки, включая MySQL-ошибку дублирования email (ER_DUP_ENTRY), и возвращает унифицированные JSON-ответы.

## Для чего нужна

Проект подходит как учебный и как базовый каркас backend-сервиса для:

- Формы регистрации пользователей — полный цикл от валидации входных данных до безопасного хранения в базу с хешированием пароля, готовое к встроению в любую веб-форму.
- Мини-CRM и внутренних административных панелей — управление списком пользователей с поиском по email и пагинацией, удобно для отслеживания зарегистрированных пользователей.
- Любого сервиса, где нужен простой REST API для работы с пользователями — базовая архитектура, которую легко расширить новыми endpoints и функциями под конкретные требования.

## Где можно использовать

- Локальная разработка и тестирование API — запустить сервер локально на localhost:3000, тестировать запросы через curl или Postman без развёртывания на боевом сервере.
- Учебные практикумы по Node.js, Express и MySQL — обучающийся материал с разделением на валидацию, сервисный слой, маршруты и middleware, демонстрирующей лучшие практики архитектуры.
- База для более крупного backend-проекта — стартовая точка, на которой можно нарастить авторизацию, другие ресурсы, логирование и мониторинг.
- Интеграция с фронтендом на React, Vue, Angular, мобильными приложениями или внешними системами через HTTP — готовый REST API, который фронтенд может вызывать через fetch или axios.

## Технологии и языки

- JavaScript (Node.js): серверная логика и API — основной язык для всей бизнес-логики, обработки запросов и взаимодействия с БД.
- SQL: схема таблиц и работа с MySQL — язык запросов для работы с пользовательскими данными, параметризованные запросы для защиты от SQL-инъекций.
- JSON: API-ответы и Postman-коллекция — стандартный формат обмена данными между сервером и клиентом, удобен для REST API.
- dotenv: конфигурация через переменные окружения — безопасное управление конфиденциальными данными (host БД, пароли) без попадания в репозиторий.
- bcrypt: хеширование паролей — криптографическая библиотека для необратимого преобразования паролей с солью, предотвращающая компрометацию в случае утечки БД.

## Структура проекта

- src/app.js: точка входа сервера и подключение middleware/маршрутов — главный файл, инициализирует Express, подключает все маршруты и middleware, запускает сервер на указанном порту.
- src/routes/users.js: маршруты users — определяет endpoints POST /users и GET /users с вызовами бизнес-логики.
- src/services/users.js: бизнес-логика пользователей — функции registerUser (хеширование и сохранение) и listUsers (получение с фильтром), разделение логики от маршрутов.
- src/validators/users.js: валидация входных данных — функция validateUser проверяет формат email и минимальную длину пароля, возвращает объект с ошибками.
- src/middlewares/errorHandler.js: централизованный обработчик ошибок — перехватывает все ошибки из маршрутов, обрабатывает специфичные DB-ошибки и возвращает унифицированные JSON-ответы.
- db/index.js: подключение к MySQL и запросы к базе — pool с параметрами из .env, функции createUser и getUsers с параметризованными SQL-запросами.
- db/schema.sql: SQL-схема таблицы users — DDL-скрипт для создания базы и таблицы с полями id, email, password_hash, created_at.
- docs/postman_collection.json: готовые запросы для тестирования API — Postman-коллекция с примерами валидных и невалидных запросов для быстрого тестирования endpoints.

## Требования

- Node.js 18+ — используется для запуска JavaScript-кода на сервере, обеспечивает асинхронность через async/await.
- npm — менеджер пакетов для установки зависимостей (express, bcrypt, dotenv, mysql2).
- MySQL 8+ (или совместимая версия) — СУБД для хранения таблицы users с данными пользователей, требует запущенного демона на хосте и порте из .env.

## Установка и запуск

1. Установить зависимости:

```bash
npm install
```
Загружает express, bcrypt, dotenv, mysql2 и другие пакеты из package.json.

2. Создать файл .env в корне проекта:

```env
PORT=3000
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASS=root
DB_NAME=prakse_db
```
Файл содержит конфиденциальные данные доступа к БД, не коммитится в гит (.env указан в .gitignore).

3. Импортировать схему базы данных:

```bash
mysql -u root -p < db/schema.sql
```
Создаёт базу prakse_db и таблицу users с необходимой структурой. Требует доступа к MySQL с правами создания БД.

4. Запустить сервер:

```bash
npm start
```
Выполняет команду из package.json: node src/app.js. Сервер стартует на порту из .env (по умолчанию 3000).

Если команда npm start не используется, можно запустить напрямую:

```bash
node src/app.js
```
Прямой запуск без посредника npm.

## Примеры использования API

Проверка сервера:

```bash
curl http://localhost:3000/health
```
Возвращает: `{"status":"ok","timestamp":"2026-03-30T12:34:56.789Z"}` — подтверждает, что сервер запущен и доступен.

Регистрация пользователя:

```bash
curl -X POST http://localhost:3000/users \
	-H "Content-Type: application/json" \
	-d '{"email":"user@example.com","password":"password123"}'
```
Возвращает: `{"id":1}` — успешно создан пользователь, или ошибка валидации/дублирования.

Получение пользователей:

```bash
curl "http://localhost:3000/users?page=1&limit=10&email=user"
```
Возвращает JSON с массивом пользователей, отфильтрованных по email, с указанной пагинацией.

## Как интегрировать в свой проект

1. Подключить текущий сервис как отдельный backend-микросервис — уже готовый REST API, может работать в отдельном контейнере или на отдельном сервере, фронтенд вызывает его endpoints.
2. Настроить переменные окружения для вашей базы MySQL — скопировать .env, изменить DB_HOST, DB_USER, DB_PASS, DB_NAME под вашу инстанцию БД.
3. Вызывать endpoint users из вашего frontend или другого backend — например, fetch("http://api-server:3000/users", {method: "POST", body: JSON.stringify({email, password})}).
4. При необходимости расширить схему users и сервисный слой в src/services — добавить новые поля в таблицу, новые функции в services для бизнес-логики.
5. Добавить авторизацию (например, JWT) поверх текущей регистрации — защитить endpoints через middleware проверки токенов, реализовать логин по email/пароль.

## Где хранится база данных
 — адрес хоста с MySQL (localhost для локального разворота, IP/domain для remote).
- DB_PORT — порт MySQL (обычно 3306, но может отличаться).
- DB_USER — username для подключения к MySQL.
- DB_PASS — пароль для подключения.
- DB_NAME — имя базы данных (prakse_db по умолчанию).

Внутри репозитория хранится только SQL-схема в файле db/schema.sql, а реальные данные находятся в вашей MySQL-инстанции на диске сервера БД. Это позволяет вести разработку локально, а продакшн использовать отдельную БД
